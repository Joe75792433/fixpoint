/ 使用方法 /
・コンパイル
    g++ -std=c++17 -o program source.cpp
・起動
    ログは時刻順になっていること
    (課題1) program < input.txt > output.txt
    (課題2,4) program N < input.txt > output.txt
    (課題3) program N m t < input.txt > output.txt

/ 出力フォーマット /
・課題1,2,4
    アドレス,発生時刻,解消時刻
・課題3
    故障/過負荷,アドレス,発生時刻,解消時刻
読み込んだログの終了時点で解消されていない場合、解消時刻は"-"と表記する

/ 説明 /
・共通
    最後に読み込んだ時にpingがタイムアウトであったサーバのアドレスと、そのサーバでいつからタイムアウトが続いているかを保持しながらログを順に読み込む
    タイムアウトが解消されたら出力を行う
    ログファイルの最後まで到達して故障が解消しなかったサーバも出力する
・課題2
    何回連続でタイムアウトしているかの情報も記録し、N未満であれば出力しないようにする
・課題3
    各サーバについて直近m回(タイムアウトを除く)のping値とその和を保持しておき、和がmt以上となったら過負荷状態が発生したとする
    m回のping値の和がmt未満となったら過負荷状態が解消されたとして出力する
    過負荷状態にあるサーバの管理は故障と同様に行う
・課題4
    各サーバが本来のアドレスとは別に、サブネット部を0に置き換えたアドレスも持っているものとして、そのアドレスにも同じ処理を行う
